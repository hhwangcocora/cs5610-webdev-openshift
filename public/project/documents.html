Welcome to Pomodoro

Pomodoro is the website to manage projects and track time spent on each task of the project. In this site, user is able to add new project, add new task, track time, view time statistics, etc.

Following is a brief introduction about each pages:

Navigation

Navigation sits on top of the page, and provides user the functions to login/logout, and register as a new user. It also displays the tabs for user to switch between different pages.

Home page

Home page displays a carousel.

Timer page

Timer page provides user a way to track time spent on a project's task. User need to select a project and a task first before start the timer. There is also a table displaying recent activities below the page.

Projects page

Projects page displays all information about projects. On this page, user can view the projects he/she currently involved in, all projects, and also can also create a new project. User can join/un-join a project. When click the "tasks page" button, user can see information related to the project, like brief introduction, currently involved users, all tasks. On this page, user can also create a new task, own/un-own a task, and view the statistics of a task.

Profile page

On profile page, user can modify his/her profile, including picure url and reset password.

Users page

Users page lists all users in this system. It displays all information related to each user, like picture, user name, full name, email, gender.

System Architecture

I used MEAN stack for this website, with frontend being HTML/CSS/AngularJS, backend as Node.JS/Express/Mongoose, and Mongo database.

In frontend, I used Bootstrap to make the GUI responsive. AngularJS $route is used to switch between different pages. To communicate with backend server, a httpService is created as an interface to communicate through http get/post.

In backend, I used Express to start the web server and accepts different kinds of endpoints. It uses json parser to parse request, passport to do local authentication. Mongoose is used to manipulate Mongo database.

Finally, Mongo

database serves as the database. Please see following section for detailed database tables desgin.

Database design

The database is consisted with 4 tables: user, project, task and record.
